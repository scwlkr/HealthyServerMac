# Buoy Brand System

## Brand Foundation

**Purpose**: Keep a Mac quietly ready for work.

**Vision**: A machine utility that feels as small and dependable as the system tools it builds on.

**Mission**: Give developers and operators a direct way to switch a Mac into a safe, server-friendly power profile without ceremony.

**Values**:

1. Restraint: Add only what makes the machine more usable.
2. Clarity: Every action should map to something concrete on macOS.
3. Reversibility: Restore is as important as enable.
4. Calm: The product should feel quiet, not gadget-like.

**Personality**:

- Direct
- Quiet
- Precise
- Native

**Brand Promise**: Buoy keeps your Mac ready without turning it into a science project.

## Positioning

**Target user**: developers, operators, homelab users, and technical founders using a Mac as a lightweight always-on workstation or utility host.

**Differentiation**:

- no dashboard
- no daemon-heavy architecture
- no hidden behavior
- native install and plain CLI fallback

**Tagline**: Keep your Mac ready.

## Visual Identity

### Color System

- Primary ink: `#142433`
- Tide blue: `#1E576F`
- Mist green: `#A8D4C9`
- Paper: `#F6F7F5`
- Stone: `#D8DEDA`
- Night: `#0C1318`

### Accessibility

- Primary text uses `#142433` on `#F6F7F5`
- Dark mode uses `#F6F7F5` on `#0C1318`
- Accent usage stays limited to highlights and actions

### Typography

- Primary: SF Pro
- Secondary: SF Mono

System typography is a deliberate brand choice here. It keeps the app fast, native, and visually restrained.

## Naming Rules

- Product: `Buoy`
- CLI: `buoy`
- Legacy alias: `healthyservermac`
- Internal types use `Buoy*`
- Docs use “Buoy mode” rather than “server mode” unless describing the specific switch

## Voice

### Voice Traits

- Short
- Technical
- Unemotional
- Recoverable

### Preferred Copy Pattern

- state the action
- state the system effect
- state the restore path when relevant

### Avoid

- hype language
- vague “optimize” claims
- phrases that imply unsupported magic

## Brand Protection

### Usage

- never pair the product with bright novelty colors
- never use playful mascots or emoji in product UI
- keep release notes and README copy operational, not promotional

### Monitoring

- README, app labels, and CLI help must use the same naming
- release assets should ship both `buoy` and `healthyservermac`
